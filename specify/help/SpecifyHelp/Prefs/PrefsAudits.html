Auditing

The auditing table in Specify can be configured to track field-level changes in the database and queried to produce a log of database transactions.

The Auditing table is accessible to users with Admin and Manager level privileges. It is not accessible in the Specify application by default and must first be enabled and configured in the User Preferences dialog:

  • For Mac: Choose Specify > Preferences > Auditing.
  • For Windows and Linux: Choose Edit > Preferences > Auditing.

Auditing Preferences

 

  • Enable Auditing allows the table to be accessible/unhidden in the Query Builder tool.

  • Audit Field Value Changes allows changes made at the field level to be accessible. This captures both the original value and the new value within the field. Capturing this data at the field-level will increase the database size and requires some processing, although in performance testing it was not noticeable.

  • Months to Save Audits sets the amount of time changes are saved and therefore accessible.

NOTE:  A value of '0' will log changes indefinitely, or until they are deleted within the system.

NOTE: Specify has previously shipped with an Audit table and existing databases will have Audit data stored starting at the time of the database creation. Setting the value for Months to Save Audits to a value less than the age of your database will result in the Audit Log deleting records older than the value the next time that Specify is started. This will process in the background and may cause a noticeable lag in the Specify application the first time old records are deleted.
